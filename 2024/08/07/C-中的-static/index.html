    一些 static 的觀念(尚未整理的筆記)

    static 怎麼用？

    static 是一個 C# 中的修飾字，主要用在「類別」上，但是是用在類別的哪裡？
    可以用在類別本身，也可以用在類別中的成員。

    當 static 用在類別身上時

    當一個類別在 C# 中被宣告為靜態類別時，有一些特性和限制：

    1. 無法實例化：靜態類別不能被實例化，這意味著不能使用 new 關鍵字創建該類別的物件。
    2. 成員只能是靜態成員：靜態類別只能包含靜態成員，如靜態方法、靜態屬性，不能包含非靜態成員。
    3. 本身有密封性（Sealed）：這意味著不能被繼承。
    4. 不能有非靜態建構函式：但可以有靜態建構函式。
    5. 不能實作介面：由於靜態類別不能被實例化，所以它不能實作任何介面。
    6. 直接透過類別名稱存取：靜態類別的成員可以直接透過類別名稱來存取，不需要實例化。
    7. 生命週期：靜態類別的成員在應用程式域的整個生命週期內都存活。

    以下是一個靜態類別的例子：

    public static class MathUtilities
{
    public static readonly double PI = 3.14159;

    public static double CalculateCircleArea(double radius)
    {
        return PI * radius * radius;
    }

    public static int Add(int a, int b)
    {
        return a + b;
    }

    static MathUtilities()
    {
        // 靜態建構函式，在類別第一次被使用時會執行
        Console.WriteLine("MathUtilities initialized");
    }
}


class Program
{
    static void Main(string[] args)
    {
      // 使用靜態類別，不用 new 出來，直接使用
      double area = MathUtilities.CalculateCircleArea(5);

      int sum = MathUtilities.Add(3, 4);

      Console.WriteLine($"Circle Area: {area}");
      Console.WriteLine($"Sum: {sum}");
    }
}

/*
  輸出結果：
  MathUtilities initialized
  Circle Area: 78.53975
  Sum: 7
*/

    當 static 用在類別中的成員時

    當一個非靜態類別中包含靜態成員時，有一些特別的情況要考慮：

    1. 類別中含有靜態成員跟非靜態成員：
      靜態成員屬於類別本身，而非靜態成員則屬於實例化後的物件。
    2. 靜態成員的存取：
      靜態成員可以直接通過類別名稱存取，不需要創建類別的實例。
      非靜態成員則需要實例化後才可以存取。
    3. 記憶體管理：
      靜態成員在應用程式啟動時就會被分配到記憶體，並在整個應用程式生命週期中存在。
      非靜態成員只有在實例化後，成為物件時才分配記憶體，並隨著物件被銷毀而釋放。
    4. 靜態成員會被共享：
      靜態成員會在所有實例化後的物件之間被共享，可以用來存儲或處理需要共用的數據。
    5. 初始化順序：
      靜態成員在類別第一次被使用時就會初始化。
      如果類別有靜態構造函數(建構子)，它會在任何靜態成員被存取之前，或任何實例被創造之前執行。
    6. 在實例中可以被存取：
      非靜態方法可以直接存取靜態成員，但靜態方法不能直接存取非靜態成員。

    以下是一個靜態成員的例子：

    public class Counter
{
    // 靜態成員
    private static int totalCount = 0;
    public static int TotalInstances => totalCount;

    // 實例成員
    public int ID { get; private set; }

    // 靜態構造函數
    static Counter()
    {
        Console.WriteLine("Static constructor called");
    }

    // 實例構造函數
    public Counter()
    {
        ID = ++totalCount;
        Console.WriteLine($"Instance {ID} created");
    }

    // 靜態方法
    public static void ResetCount()
    {
        totalCount = 0;
    }

    // 實例方法
    public void DisplayInfo()
    {
        Console.WriteLine($"This is instance {ID} out of {totalCount} total instances");
    }
}

class Program
{
    static void Main(string[] args)
    {
        // 靜態建構子會先被呼叫後，才執行靜態成員，這邊呼叫到的是「TotalInstances」這個靜態屬性
        Console.WriteLine($"Total instances before creating any: {Counter.TotalInstances}");

        Counter c1 = new Counter();
        Counter c2 = new Counter();

        c1.DisplayInfo();
        c2.DisplayInfo();

        Console.WriteLine($"Total instances: {Counter.TotalInstances}");

        Counter.ResetCount();
        Console.WriteLine($"After reset, total instances: {Counter.TotalInstances}");
    }
}

/*
  輸出結果：
  Static constructor called
  Total instances before creating any: 0
  Instance 1 created
  Instance 2 created
  This is instance 1 out of 2 total instances
  This is instance 2 out of 2 total instances
  Total instances: 2
  After reset, total instances: 0
*/
