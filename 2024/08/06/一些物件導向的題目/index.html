    2. 定義函式多載（Function Overload）並給出一個使用不同參數的方法多載例子

    函式多載是指在一個類別中，定義多個名稱相同但參數類型不同的函式。
    這些函式可以有不同的參數類型、參數數量或參數順序。編譯器會根據呼叫時提供的參數來決定使用哪一個函式。

    public class Calculator
{
    // 兩個整數相加
    public int Add(int a, int b)
    {
        return a + b;
    }

    // 三個整數相加
    public int Add(int a, int b, int c)
    {
        return a + b + c;
    }

    // 兩個浮點數相加
    public double Add(double a, double b)
    {
        return a + b;
    }
}

class Program
{
    static void Main(string[] args)
    {
        Calculator calc = new Calculator();

        int result1 = calc.Add(5, 3);           // 呼叫 int Add(int a, int b)
        int result2 = calc.Add(5, 3, 2);        // 呼叫 int Add(int a, int b, int c)
        double result3 = calc.Add(2.5, 3.7);    // 呼叫 double Add(double a, double b)
    }
}

    3. 建構函式在 C# 中的用途是什麼？提供一個帶有參數的建構函式的例子

    建構函式(建構子)在 C#中的主要用途是:

    1. 設定預設值: 當創建一個新的物件實例時，若沒有給與初始化的參數，則會自動將屬性套上預設值。
    2. 設定初始狀態: 當創建一個新的物件實例時，若有給與初始化的參數，則將這些參數賦予到新物件的屬性上。
    3. 準備空間: 建構函式可以替物件分配出必要的記憶體空間。
    4. 執行必要的設定: 負責所有物件「開始運作」前的準備工作，包括檢查某些條件是否滿足，進行一些初始化的計算等等。

    以下是一個帶有參數的建構函式的例子:

    public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }

    // 帶有參數的建構函式
    public Person(string name, int age)
    {
        Name = name;
        Age = age;
    }
}

class Program
{
    static void Main(string[] args)
    {
        // 使用建構函式創建 Person 物件
        Person person = new Person("Alice", 30);

        Console.WriteLine($"Name: {person.Name}, Age: {person.Age}");
    }
}

    4. 提供一個例子，展示如何在 C# 中覆寫父類別

    這個例子將展示如何使用 override 關鍵字來覆寫父類別的方法。

    // 父類別
public class Animal
{
    public string Name { get; set; }

    public Animal(string name)
    {
        Name = name;
    }

    public virtual void MakeSound()
    {
        Console.WriteLine("動物發出聲音");
    }
}

// 子類別
public class Dog : Animal
{
    public Dog(string name) : base(name)
    {
        /*  這段程式碼是 Dog 類別的 public 建構函式，
            它接受一個字串參數 name，而「: base(name)」這部分是關鍵。
            :base() 語法用於呼叫父類別的建構函式。
            在這個例子中，它呼叫 Animal 類別的建構函式，並將 name 參數傳遞給父類別。

          為什麼要這樣做?
            1. 繼承性: Dog 類別繼承自 Animal 類別，所以它應該初始化從 Animal 繼承的所有屬性。
            2. 重用代碼: 通過呼叫父類別的建構函式，我們可以避免重複編寫相同的初始化程式碼。
            3. 保持一致性: 確保 Dog 物件的 Name 屬性(繼承自 Animal)且被正確初始化。 
        */
    }

    // 覆寫父類別的方法
    public override void MakeSound()
    {
        Console.WriteLine($"{Name} 汪汪叫");
    }
}

class Program
{
    static void Main(string[] args)
    {
        Animal genericAnimal = new Animal("未知動物");
        Animal myDog = new Dog("小白");

        genericAnimal.MakeSound(); // 輸出：動物發出聲音
        myDog.MakeSound(); // 輸出：小白 汪汪叫
    }
}

    5. 舉例說明如何使用 this 和 base 關鍵字

    public class Animal
{
    public string Name;

    public Animal(string name)
    {
        this.Name = name; // this.Name = Animal.Name，在這邊 this 就是 Animal( this 會指向當前的物件 )。
    }

    public virtual void Speak()
    {
        Console.WriteLine($"{Name} 發出聲音");
    }
}

public class Dog : Animal
{
    public string Breed;

    public Dog(string name, string breed) : base(name) // 呼叫父類別 Animal 的建構函式，並傳遞 name 參數過去。
    {
        this.Breed = breed;
    }

    public override void Speak()
    {
        base.Speak(); // 呼叫父類別 Animal 的 Speak 方法。
        Console.WriteLine($"{this.Name} 汪汪叫");
    }
}

class Program
{
    static void Main(string[] args)
    {
        Dog myDog = new Dog("小白", "哈士奇");
        myDog.Speak();  // 輸出：小白 發出聲音
                        // 輸出：小白 汪汪叫
    }
}

    在這個例子中：

    1. this 的用法：
    • 在 Animal 類別中：this.Name = name; 這裡使用 this 來區分屬性 Name 和參數 name。
    • 在 Dog 類別中：this.Breed = breed; 同樣用於區分屬性和參數。
    • 在 Dog 類的 Speak 方法中：this.Name 明確指出我們正在使用當前物件的 Name 屬性。
    1. base 的用法：
    • 在 Dog 的建構函式中：:base(name) 呼叫父類別 Animal 的建構函式，並傳遞 name 參數。
    • 在 Dog 的 Speak 方法中：base.Speak(); 呼叫父類別 Animal 的 Speak 方法。

    7. 如何在 C# 中使用介面來實現多型？提供一個具體的例子

    這個例子展示了多型的運用

    1. 定義一個 IShape 介面，它只有一個方法 CalculateArea()。
    2. 創建兩個類別 Circle 和 Square，它們都實作了 IShape 介面。
    3. 這兩個類別都實作了 CalculateArea() 方法：
      Circle 使用圓形面積公式 πr²
      Square 使用正方形面積公式 s²
    4. 在 Main 方法中，實例化一個圓形物件和一個正方形物件，但都將它們賦值給 IShape 型別的變數。
    5. PrintArea() 方法接受一個 IShape 型別的參數。這意味著它可以接收任何實作 IShape 介面的物件。
    6. 在 Main 方法中，呼叫 PrintArea() 方法，並傳遞 IShape 型別的變數來印出面積。
    7. 當我們呼叫 PrintArea() 方法時，它不用知道傳入的是圓形還是正方形，只需要知道這個物件實作了 IShape 介面，因此一定有 CalculateArea() 方法即可。
    // 定義一個介面
public interface IShape
{
    double CalculateArea();
}

// 實作介面的圓形類別
public class Circle : IShape
{
    public double Radius { get; set; }

    public Circle(double radius)
    {
        Radius = radius;
    }

    public double CalculateArea()
    {
        return Math.PI * Radius * Radius;
    }
}

// 實作介面的正方形類別
public class Square : IShape
{
    public double Side { get; set; }

    public Square(double side)
    {
        Side = side;
    }

    public double CalculateArea()
    {
        return Side * Side;
    }
}

class Program
{
    static void PrintArea(IShape shape)
    {
        Console.WriteLine($"面積是: {shape.CalculateArea():F2}");
    }

    static void Main(string[] args)
    {
        IShape circle = new Circle(5);
        IShape square = new Square(4);

        PrintArea(circle);  // 輸出圓形的面積
        PrintArea(square);  // 輸出正方形的面積
    }
}

    8. 在 C# 中，當一個類別被聲明為 sealed 時，其成員（方法、屬性等）還可以被覆寫嗎？解釋原因

    在 C# 中，當一個類別被宣告為 sealed 時，這個類別無法被繼承，
    也因為沒辦法被繼承，所以其成員（方法、屬性等）無法被其他類別複寫。

    9. 解釋靜態修飾詞 static 在 C# 中何時會被使用，並給出一個例子

    在 C#中，靜態修飾詞 static 主要在以下情況下使用：

    1. 共享資源：當需要一個在所有類別實例間共享某個資源，可以使用靜態修飾詞。

    2. 工具方法：當方法不需要實例化就可以直接取用時。

    3. 常數：定義好且不會改變的值。

    下面是一個簡單的例子：

    public class MathHelper
{
    // 靜態常數
    public static readonly double PI = 3.14159;

    // 靜態變量，用於計數方法被調用的次數
    private static int calculationCount = 0;

    // 靜態方法
    public static double CalculateCircleArea(double radius)
    {
        calculationCount++; // 增加計數
        return PI * radius * radius;
    }

    // 靜態方法，用於獲取計算次數
    public static int GetCalculationCount()
    {
        return calculationCount;
    }
}

class Program
{
    static void Main(string[] args)
    {
        // 使用靜態方法，無需創建類別實例
        double area1 = MathHelper.CalculateCircleArea(5);
        Console.WriteLine($"圓的面積: {area1}");

        double area2 = MathHelper.CalculateCircleArea(3);
        Console.WriteLine($"另一個圓的面積: {area2}");

        // 獲取計算次數
        int count = MathHelper.GetCalculationCount();
        Console.WriteLine($"計算次數: {count}");
    }
}

/*
    輸出結果：
        圓的面積: 78.53975
        另一個圓的面積: 28.27431
        計算次數: 2
*/

    11. 解釋何時應該使用 readonly 屬性，並給出一個例子

    readonly 關鍵字，在 C# 中通常用在類別的屬性上，此屬性只能在初始化時指派值。
    以下是使用 readonly 的時機：

    1. 常數值：當一個值在物件生命週期內不應該變動時。
    2. 相依性注入：當一個物件在建構時被注入，之後不應被更改時。
    3. 提升效能：編譯器可以對 readonly 欄位進行最佳化。
    4. 執行緒安全：readonly 欄位在多執行緒環境中較為安全，因為它們在初始化後不會改變。
    5. 程式碼清晰度：明確表示某個欄位不應在初始化後被修改。

    以下是個簡單的例子：

    public class Student
{
    // readonly 欄位，學號不可變
    private readonly string studentId;

    // 可變欄位
    public string Name { get; set; }

    // 建構子
    public Student(string id, string name)
    {
        studentId = id;
        Name = name;
    }

    // 取得學號的方法
    public string GetStudentId()
    {
        return studentId;
    }
}

class Program
{
    static void Main(string[] args)
    {
        Student student = new Student("S001", "小明");

        Console.WriteLine($"學生姓名：{student.Name}");
        Console.WriteLine($"學號：{student.GetStudentId()}");

        // 可以更改名字
        student.Name = "大明";
        Console.WriteLine($"更改後的姓名：{student.Name}");

        // 以下程式碼會導致編譯錯誤，因為 studentId 是 readonly
        // student.studentId = "S002"; // 錯誤
    }
}

/*
    輸出結果：
        學生姓名：小明
        學號：S001
        更改後的姓名：大明
*/

    在這個例子中：

    1. studentId 被宣告為 readonly，表示它只能在建構子中設定，之後就不能更改。
    2. Name 是一個普通的屬性，可以隨時更改。
    3. 建構子接受學號和姓名，並設定相應的欄位。
    4. GetStudentId() 方法用來取得學號，因為 studentId 是私有的。
    5. 在 Main 方法中，我們可以建立 Student 的實例，讀取學號，更改姓名，但不能更改學號。

    這個例子簡單地展示了 readonly 的用途：

    • 確保重要的識別資訊（如學號）在物件建立後不會被更改。
    • 同時允許其他資訊（如姓名）保持可變。

    12. 示範如何在 C# 中重載一個類別的建構函式並解釋其目的

    以一個簡單的 Person 類別為例。

    public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
    public string City { get; set; }

    // 建構函式 1：沒有參數
    public Person()
    {
        Name = "未知";
        Age = 0;
        City = "未知";
    }

    // 建構函式 2：有名字參數
    public Person(string name)
    {
        Name = name;
        Age = 0;
        City = "未知";
    }

    // 建構函式 3：有名字和年齡參數
    public Person(string name, int age)
    {
        Name = name;
        Age = age;
        City = "未知";
    }

    // 建構函式 4：有名字、年齡和城市參數
    public Person(string name, int age, string city)
    {
        Name = name;
        Age = age;
        City = city;
    }
}

class Program
{
    static void Main(string[] args)
    {
        Person person1 = new Person();
        Person person2 = new Person("小明");
        Person person3 = new Person("小華", 25);
        Person person4 = new Person("小李", 30, "台北");

        Console.WriteLine($"Person1: {person1.Name}, {person1.Age}, {person1.City}");
        Console.WriteLine($"Person2: {person2.Name}, {person2.Age}, {person2.City}");
        Console.WriteLine($"Person3: {person3.Name}, {person3.Age}, {person3.City}");
        Console.WriteLine($"Person4: {person4.Name}, {person4.Age}, {person4.City}");
    }
}

    解釋建構函式多載的目的：

    1. 彈性初始化：
      允許以不同的方式初始化物件，根據可用的資訊量。
      使用者可以選擇最適合他們需求的建構函式。

    2. 預設值處理：
      對於未提供的資訊，可以設定合理的預設值。

    3. 程式碼重用：
      較複雜的建構函式可以呼叫較簡單的建構函式，減少重複程式碼。

    4. 向後相容性：
      在維護或擴展類別時，可以新增建構函式而不影響現有的程式碼。

    5. 提高可讀性：
      明確地表示物件可以用不同的方式被建立，使程式碼更易懂。

    參數驗證：
    可以在不同的建構函式中執行不同的參數驗證邏輯。
    多載建構函式讓類別的使用更加靈活，能夠適應不同的使用場景。這種方法在建立複雜物件或當物件的初始化可能有多種形式時特別有用。

    14. 什麼是 sealed 類別，並解釋其如何用來限制繼承。

    在 C# 中，sealed 類別是一種特殊的類別，它被設計用來防止其他類別繼承它。
    當一個類別被宣告為 sealed 時，它就成為了繼承鏈的終點。

    1. sealed 類別的定義：使用 sealed 關鍵字來宣告一個類別
    public sealed class SealedClass
{
    // 類別成員
}
    1. 限制繼承的方式：
      當一個類別被宣告為 sealed，其他類別就不能繼承它。
      如果嘗試繼承一個 sealed 類別，編譯器會產生錯誤，如下例子：
    public sealed class SealedClass
{
    public void SealedMethod()
    {
        Console.WriteLine("這是密封類別中的方法");
    }
}

// 以下程式碼會導致編譯錯誤
// public class AttemptToInherit : SealedClass
// {
//     // 無法繼承 SealedClass
// }

    1. sealed 類別的用途：
      防止意外或不當的繼承。
      確保類別的行為不被修改或擴展。
      在某些情況下可以提高性能，因為編譯器可以進行特定的優化。

    2. 注意事項：
      sealed 類別仍然可以繼承其他非 sealed 類別。
      sealed 類別可以實作介面。
      類別中的個別方法也可以被宣告為 sealed，以防止在子類別中被覆寫。

    3. 使用場景：
      當你設計一個不應該被繼承的類別時，例如工具類別或特定的業務邏輯類別。
      在安全性要求高的場景中，防止類別被繼承和修改，如下例子：

    public sealed class SecurityManager
{
    public void PerformSecureOperation()
    {
        Console.WriteLine("執行安全操作");
    }
}

class Program
{
    static void Main(string[] args)
    {
        SecurityManager manager = new SecurityManager();
        manager.PerformSecureOperation();
    }
}

    總結來說，sealed 類別提供了一種方法來限制類別的繼承，這在某些設計情境中是很有用的，特別是當你需要確保類別的行為不被更改或擴展時。但是使用 sealed 類別也會限制程式的靈活性，所以在決定使用時需要權衡利弊。

    15. 解釋 C# 中繼承的概念並給出一個簡單範例。

    繼承是物件導向程式設計中的一個核心概念。
    在 C# 中，繼承允許我們創建一個新的類別（稱為子類別或衍生類別），該類別基於一個現有的類別（稱為父類別或基礎類別）。
    子類別繼承了父類別的屬性和方法，同時可以添加自己的新成員或覆寫父類別的成員。

    繼承的主要優點：

    1. 代碼重用：避免重複編寫相同的代碼。
    2. 建立層次結構：允許創建更專門化的類別。
    3. 多型性：允許使用父類別型別來引用子類別物件。

    簡單的範例來說明繼承：

    // 基礎類別（父類別）
public class Shape
{
    public string Color { get; set; }

    public Shape(string color)
    {
        Color = color;
    }

    public virtual void Draw()
    {
        Console.WriteLine($"畫一個{Color}色的形狀");
    }
}

// 衍生類別（子類別）
public class Circle : Shape
{
    public Circle(string color) : base(color)
    {
    }

    public override void Draw()
    {
        Console.WriteLine($"畫一個{Color}色的圓形");
    }
}

class Program
{
    static void Main(string[] args)
    {
        Shape shape = new Shape("紅");
        shape.Draw();  // 輸出：畫一個紅色的形狀

        Circle circle = new Circle("藍");
        circle.Draw();  // 輸出：畫一個藍色的圓形

        // 多型示範
        Shape shapeCircle = new Circle("綠");
        shapeCircle.Draw();  // 輸出：畫一個綠色的圓形
    }
}

    16. 解釋抽象類別，並給出一個抽象類別的例子。

    抽象類別是一種特殊的類別，它不能被直接實例化，主要用於定義其他類別的共同特徵。
    抽象類別可以包含抽象方法（沒有實作的方法）和具體方法（有實作的方法）。

    抽象類別的主要特點：

    1. 使用 abstract 關鍵字宣告。
    2. 不能被直接實例化。
    3. 可以包含抽象方法和具體方法。
    4. 子類別必須實作所有抽象方法，除非子類別也是抽象類別。
    5. 可以包含屬性和欄位。

    以下是一個抽象類別的例子：

    // 抽象類別
public abstract class Shape
{
    // 屬性
    public string Color { get; set; }

    // 建構函式
    protected Shape(string color)
    {
        Color = color;
    }

    // 抽象方法（沒有實作）
    public abstract double CalculateArea();

    // 具體方法（有實作）
    public virtual void DisplayInfo()
    {
        Console.WriteLine($"這是一個{Color}色的形狀");
    }
}

    17. 描述如何使用抽象方法強制子類別提供特定功能的實現。

    // 抽象類別
public abstract class Shape
{
    // 屬性
    public string Color { get; set; }

    // 建構函式
    protected Shape(string color)
    {
        Color = color;
    }

    // 抽象方法（沒有實作）
    public abstract double CalculateArea();

    // 具體方法（有實作）
    public virtual void DisplayInfo()
    {
        Console.WriteLine($"這是一個{Color}色的形狀");
    }
}

// 繼承抽象類別的具體類別
public class Circle : Shape
{
    public double Radius { get; set; }

    public Circle(string color, double radius) : base(color)
    {
        Radius = radius;
    }

    // 實作抽象方法，必須實作 CalculateArea() 這個抽象方法。
    public override double CalculateArea()
    {
        return Math.PI * Radius * Radius;
    }

    // 選擇性覆寫具體方法
    public override void DisplayInfo()
    {
        base.DisplayInfo();
        Console.WriteLine($"這是一個半徑為 {Radius} 的圓");
    }
}

class Program
{

    {
        // Shape shape = new Shape("紅色"); // 這行會造成編譯錯誤，因為抽象類別不能被實例化

        Circle circle = new Circle("藍色", 5);
        circle.DisplayInfo();
        Console.WriteLine($"圓的面積是: {circle.CalculateArea()}");
    }
}
/*

        這是一個藍色色的形狀
        這是一個半徑為 5 的圓

    在這個例子中：

    1. Shape 是一個抽象類別：
      它有一個 Color 屬性。

    2. Circle 類別繼承自 Shape：

      可以選擇性地覆寫 DisplayInfo() 方法。

      我們不能直接實例化 Shape。

    20. 介面與抽象類別有什麼不同？

    1. 定義方式：
      抽象類別：使用 abstract class 關鍵字定義。
      介面：使用 interface 關鍵字定義。

    2. 實作：
      抽象類別：可以包含抽象方法和具體方法。
      介面：在 C# 8.0 之前，只能包含抽象方法。從 C# 8.0 開始，可以包含預設實作。

    3. 繼承：
      抽象類別：一個類別只能繼承一個抽象類別。
      介面：一個類別可以實作多個介面。

    4. 建構函式：
      抽象類別：可以有建構函式。
      介面：不能有建構函式。

    5. 存取修飾詞：
      抽象類別：可以使用各種存取修飾詞（public, protected, internal 等）。
      介面：成員默認是 public，不能使用其他存取修飾詞。

    6. 用途：
      抽象類別：用於表示 “是一種” (is-a) 關係，提供基本實作。
      介面：用於定義 “能做什麼” (can-do) 的規範，專注於行為。
