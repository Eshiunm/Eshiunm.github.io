    • JavaScript 的型別

    在 JavaScript 的世界裡，只有兩大型別，分別是 Primitive Type（原始型別）和 Object Type（物件型別）。

    原始型別的種類如下：

    • Boolean
    • Number
    • String
    • BigInt
    • Symbol
    • Null
    • Undefined

    以下種類皆為物件型別：

    • Array
    • Object
    • Function

    除了原始型別以外，其餘都可以歸類在物件型別，例如陣列、物件、函式都屬於物件型別。

    JavaScript 共有 7 種 「原始型別，其餘的都屬於 「物件型別」。

    如何區分原始型別與物件型別？

    原始型別無法增加屬性，物件型別可以增加屬性

    原始型別：

    let string = "我是小明";
string.name = "我是字串";
console.log(string.name); // undefined，無法新增屬性
console.log(string); // 我是小明

    物件型別：

    // 對物件新增屬性
let object = {
  property: "我是物件",
}
object.name = "我是一個屬性";
console.log(obj.name); // "我是一個屬性"

// 對陣列新增屬性（實戰中請不要對陣列這樣做)
let arr = [1, 2, 3];
arr.name = "我是一個陣列";
console.log(arr.name); // "我是一個陣列"
console.log(arr)// [1, 2, 3, name: '我是一個陣列']

// 對函式新增屬性（實戰中請不要對函式這樣做)
function fn() {}
fn.myName = "我是一個函式"; // 函式本身就有 name 屬性，別覆蓋它
console.log(fn.myName); // "我是一個函式"
console.dir(fn); // 可以印出函式的結構

    型別判斷方式

    在大多數型別中，都可以透過 typeof 運算子來判別，是實戰中最常見用來判斷型別的手法，例如下方的一些例子：

    console.log(typeof "這是一句話"); // "string"
console.log(typeof 123); // "number"
console.log(typeof true); // "boolean"
console.log(typeof Symbol("sym")); // "symbol"
console.log(typeof 100n); // "bigint"
console.log(typeof null); // "object"，null 會被判斷成 Object 型別，這是一個 JS 的 Bug，然後官方也不打算改( 因為會牽一髮動全身 )
console.log(typeof undefined); // "undefined"
console.log(typeof []); // "object"
console.log(typeof {}); // "object"
console.log(typeof function () {}); // "function"，雖然印出 function，但還是當作物件型別看待
